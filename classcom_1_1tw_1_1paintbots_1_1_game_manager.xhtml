Detailed Description

The GameManager is the core class of the game. It creates all Entities and manages their behavior. It also calls the update routine and is responsible for rendering. The GameManager implements the singleton pattern.

◆ getMoveOder()

ArrayList< Integer > com.tw.paintbots.GameManager.getMoveOder ( )
inline

In each turn the move order is cycled. E.g. if four players take part at the game in the 1st turn the order is (0,1,2,3) in the 2nd turn (1,2,3,0) in the 3rd turn (2,3,0,1) and so on. With this method you can get the order in which the players are handled. This is especially relevant for painting at the canvas (the last one decides the final color) and collecting power ups (the first one gets the power up).

Returns
An ArrayList with the indexes of the players in their move order.

◆ update()

void com.tw.paintbots.GameManager.update ( GameKey  key)
inline

The update routine that is called in the loop. This method can only be called by the PaintBotsGame class. Depending on the GameState different things happen:

  • MENU: The keyboard controls are used to select bots and levels
  • STARTTIMER: Count down 5 seconds at the start of the game.
  • GAME: Update all entities, especially the players/bots.
Parameters
keyThe hidden GameKey which is only available to the PaintBotsGame class.
